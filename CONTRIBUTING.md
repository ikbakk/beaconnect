# Contributing

Beaconnect welcomes thoughtful contributions, but every contribution must preserve the product philosophy.

## Contribution Standard

A contribution is acceptable when it:

- Supports reassurance.
- Respects mutual consent.
- Uses calm human language.
- Avoids surveillance patterns.
- Keeps the app simple.
- Does not increase notification noise.
- Does not expose technical implementation to users.

## Pull Request Template

Every PR should answer:

```md
## Why

What user problem does this solve?

## Product Principle

Which Beaconnect principle does this strengthen?

## Calm Test

- [ ] Does not increase anxiety
- [ ] Uses neutral wording
- [ ] Respects consent
- [ ] Does not add unnecessary notifications
- [ ] Does not expose technical wording

## UX

Why is this the simplest solution?

## Screens

Which screens changed?

## Non-Goals

Does this accidentally move Beaconnect toward a non-goal?

## Docs

Which docs were updated?
```

## 2× Rule

Every feature proposal must answer:

1. What can be removed because of this feature?
2. What existing principle does this strengthen?

If both answers are weak, the feature probably should not exist.
