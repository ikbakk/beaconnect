# Non-Goals

Beaconnect intentionally does not try to solve every problem related to location sharing.

## Beaconnect is NOT

### Not a Messaging App

Users already have excellent messaging applications.

Beaconnect complements them with reassurance features instead of replacing conversations.

### Not a Social Network

No feeds.

No followers.

No likes.

No public profiles.

No engagement mechanics.

### Not a Surveillance Tool

Beaconnect never encourages secretly monitoring another person.

Location sharing is always:

- mutual
- transparent
- revocable
- consent-based

### Not a Parental Monitoring Platform

The first version is designed for two consenting people.

Parent-child hierarchy may exist in the future, but it is outside v1.

### Not an Employee Tracking Solution

Beaconnect is never intended for workplace attendance, workforce management, or employee monitoring.

### Not an Emergency Response System

Beaconnect can improve peace of mind, but it must never be presented as a substitute for emergency services or personal safety systems.

### Not a Navigation App

Maps exist to provide context.

The primary experience is reassurance, not turn-by-turn navigation.

### Not a Fitness Tracker

Location data is used for reassurance.

It is not intended to measure exercise, performance, or health metrics.

### Not a Productivity Tracker

Beaconnect should never encourage users to measure another person's habits, routines, or efficiency.

### Not a Data Collection Platform

Collect only the information required to deliver the experience.

Data collection should always be intentional, explainable, and minimal.

## Product Guardrails

Before implementing any feature, ask:

- Does this reduce anxiety?
- Could this accidentally increase anxiety?
- Is it transparent?
- Does it respect mutual consent?
- Is it understandable?
- Is it worth the battery cost?
- Does it interrupt users unnecessarily?
- Are we describing system state instead of assuming human intent?
- Does it still feel like Beaconnect?
