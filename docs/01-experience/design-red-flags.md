# Design Red Flags

This document is an early warning system.

## Product Drift

Red flag:

- logo larger than partner
- map larger than partner by default
- navigation draws more attention than content
- action button becomes the main character

## Feature Creep

Reject or review immediately:

- chat
- voice calls
- photo sharing
- stories
- comments
- streaks
- achievements

## Technical Leakage

Reject user-facing words like:

- GPS
- sync
- cache
- API
- background service
- reverse geocoding
- foreground service

## Anxiety

Review anything that makes users think something is wrong without evidence.

Examples:

- flashing yellow
- urgent banners
- aggressive battery warnings
- countdowns outside Live Sharing
- repeated warnings

## Navigation Growth

If someone proposes a fourth tab, ask:

> Can this be a state instead of a destination?

## Second Primary Action

Every screen gets:

- one visual anchor
- one primary action

## Decorative Motion

Motion exists because something changed.

Not because something appeared.

## New Setting

Every new setting must answer:

> Why is the default not good enough?

## Widget Growth

The widget is not a miniature app.

If it gains scrolling, it has failed.

## Final Red Flag

If a design decision makes Beaconnect more noticeable than the relationship, stop and redesign.
