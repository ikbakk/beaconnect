# Interaction Language

## Motion Principle

Motion should maintain continuity.

Motion exists to explain change, not to impress.

## Categories

### State Motion

Used when a state changes.

Example:

```text
Normal Partner Card
→ Live Sharing Card
```

### Navigation Motion

Use standard Material transitions.

No custom spectacle.

### Feedback Motion

Used after user action.

Example:

```text
I'm Okay pressed
→ gentle compression
→ ripple
→ confirmation
```

### Context Motion

Used when context expands.

Example:

```text
Small map preview
→ larger map view
```

## Rules

- Never replace when you can transform.
- Motion follows hierarchy.
- Animate only the changing element.
- Do not animate the logo in core screens.
- Do not animate decorative backgrounds.
- Do not use confetti.
- Do not use repeated pulsing.
- Do not use attention-grabbing empty animations.

## Haptics

Use sparingly:

- Check-in: tiny confirmation.
- Live started: slightly stronger.
- Pair completed: warm confirmation.
- Error: single clear haptic.

## Sounds

No in-app sounds in v1.

Notifications use system behavior.
