# User Flows

## Flow 1 — First Open

```text
Open app
→ Welcome
→ Explain: Built for reassurance, not surveillance
→ Explain mutual sharing
→ Continue to account
```

## Flow 2 — Account Setup

```text
Create account / sign in
→ Add display name
→ Continue
```

Rules:

- Display name required.
- Profile photo omitted in v1.
- Account-based, not device-based.

## Flow 3 — Pairing

```text
User A creates invite code
→ User B enters invite code
→ Both confirm names
→ Both approve
→ Pair created
```

Rules:

- One-time code.
- Expires in 5 minutes.
- Both sides approve.
- Either side can cancel.

## Flow 4 — Permission Education

```text
Pair created
→ Explain why permission helps
→ Ask Android permission
→ Continue with clear status
```

Tone:

- Recommend, never pressure.
- Explain first.

## Flow 5 — Home

```text
Open app
→ Partner Card
→ I'm Okay
→ Recent Updates
→ Map Preview
```

## Flow 6 — Check-in

```text
Tap I'm Okay
→ small press feedback
→ send check-in
→ confirmation
→ partner receives notification
→ partner can react
```

## Flow 7 — Request Check-in

```text
Tap Request Check-in
→ Partner receives gentle notification
→ Partner chooses I'm Okay / Later
→ Requester sees calm result
```

## Flow 8 — Live Sharing

```text
Start Live
→ Choose duration
→ Optional reason
→ Start
→ Pause/End available
→ Auto-end when duration expires
```

## Flow 9 — Pause Sharing

```text
Pause sharing
→ Choose duration
→ Partner sees sharing paused
→ Resume manually or automatically
```

## Flow 10 — Updates

```text
Open Updates
→ See Today
→ See meaningful grouped events
→ View details if needed
→ End with completion message
```

## Flow 11 — Trust Center

```text
Open Settings
→ Trust Center
→ Explain → Status → Action
```

## Flow 12 — Widget

```text
Glance widget
→ understand partner status
→ tap I'm Okay if desired
→ open app only when more context is needed
```
