# UX Review Template

Use this for every screen or feature.

## Screen / Feature

Name:

## Purpose

One sentence.

## User Intent

What question is the user trying to answer?

## Information Hierarchy

Top to bottom.

## Primary Action

Only one.

## Secondary Actions

Maximum four.

## Empty State

What happens when there is no data?

## Error / Recovery State

What happens when something cannot complete?

## Calm Test

- [ ] Reduces anxiety
- [ ] Does not assume intent
- [ ] Uses neutral wording
- [ ] Gives context without pressure

## Grandma Test

- [ ] No technical jargon
- [ ] Meaning is understandable without explanation

## Gravity Test

What is the screen's emotional center?

## Android Test

Does it respect Android conventions?

## Removal Test

What happens if this screen or feature is removed?

## Decision

Approve / Reject / Revise.
