# DESIGN.md

## Purpose

Beaconnect is a reassurance-first application.

Every visual decision should reinforce calm, trust, and presence rather than excitement or engagement.

## Design Principles

### Presence over Precision

People care more about knowing someone is around than seeing precise technical information.

### Quiet by Default

Beaconnect should never compete for attention.

### Reserved Outside, Caring Inside

Beaconnect does not greet users loudly.

Its warmth appears through thoughtful interactions and human wording.

### Explain Before Configuring

Explain why.

Then show current status.

Then provide action.

### Information Earns Its Place

If information does not help answer:

> Do I need to do anything?

it probably does not belong on the primary interface.

### Progressive Disclosure

Home provides reassurance.

Updates provide context.

Details appear only when requested.

## Visual Personality

Beaconnect should feel:

- calm
- quiet
- dependable
- respectful
- human
- thoughtful

It should never feel:

- loud
- flashy
- corporate
- overly playful
- gamified
- attention-seeking

## Style

- Clean.
- Minimal.
- Slightly playful.
- Never childish.

Approximate balance:

```text
90% calm
10% charm
```

## Spacing

Comfortably breathable.

More spacious than Material defaults.

Less spacious than Samsung One UI.

## Corners

Moderately rounded.

Not as round as Samsung One UI.

Not as square as Microsoft Fluent.

## Motion

Motion exists to explain change.

Avoid:

- bouncing
- confetti
- overshoot animations
- attention-grabbing effects

## Color

Muted.

Warm.

Natural.

Avoid highly saturated colors.

Red is rare and reserved for meaningful urgency.

## Icons

Simple.

Friendly.

Rounded without becoming cartoonish.

## Illustrations

Rare.

Purposeful.

Minimal.

Avoid AI-generated visual clichés.

## Final Principle

The interface should never try to impress the user.

It should quietly become part of their life.
